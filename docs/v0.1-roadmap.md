## Cross-cutting rules

- `P:\kirei-manga` (Windows) / `~/dev/kirei-manga` (macOS) ŌĆö same repo, `master` branch.
- **Never** run `pnpm dev` for desktop/api unsolicited (global rule). Verify with `pnpm typecheck`, `pnpm build`, `pnpm test`.
- Small conventional commits per phase ŌĆö no batch commits.
- No `Co-Authored-By` on any commit.
- Renderer never talks to `api.mangadex.org` / `uploads.mangadex.org` directly. All MangaDex traffic goes through the NestJS backend (CORS + hotlinking bans).
- All MangaDex images served via `kirei-cover://` and `kirei-page://` custom protocols.
- User-Agent `KireiManga/0.1.0 (https://github.com/Shironex/kirei-manga)` on every upstream request.
- Rate limits: 220ms min interval on `api.mangadex.org` general; 1500ms on `/at-home/server/{id}`.
- Design identity per `.impeccable.md`: editorial, hairline rules, Fraunces display, bengara (iron-oxide red) as the single accent. No dashboard-card grids, no glassmorphism, no neon.

## Slice B ŌĆö Series detail + follow/unfollow

**Goal:** clicking a cover in Browse opens a series detail page with metadata, chapter list, and a follow button that persists to SQLite.

### Phases

1. **`feat(shared): add series detail payload types`**
   - Extend `packages/shared/src/types/series.ts` and/or `mangadex.ts` with a normalized `MangaDexSeriesDetail` (already partially defined) that includes synopsis, author, artist, tags[], status, year, contentRating, originalLanguage, availableTranslatedLanguages, and a chapter summary.
   - Add `ChapterListItem` type for the chapter list UI (id, volume, chapter, title, translatedLanguage, publishAt, pages, scanlationGroup, externalUrl).

2. **`feat(desktop): implement mangadex.get-series and mangadex.get-chapters`**
   - Service: call `MangaDexClient.getSeries(id)` ŌåÆ normalize to `MangaDexSeriesDetail`. Resolve cover to `kirei-cover://mangadex/...`, with `original` size option for banner.
   - `getChapters(seriesId, lang?)`: paginate `/manga/{id}/feed`, filter invalid (externalUrl set, pages=0), sort by volume+chapter ascending, dedupe by chapter number keeping first (or most-recent) translation per group.
   - Respect `offset + limit Ōēż 10000` cap; stop early once `offset >= total`.

3. **`feat(desktop): library follow/unfollow service with sqlite persistence`**
   - `LibraryService.follow(mangadexId)`: fetch MangaDex series, upsert `series` row with `source='mangadex'`, default `status='reading'`, `addedAt=now`. Return the stored `Series`.
   - `unfollow(id)`: delete row by local id.
   - `getAll()`, `getSeries(id)`: read from SQLite.
   - `updateStatus(id, status)`, `updateProgress(id, chapterId, page)`: update rows.
   - Emit `library:changed` event over Socket.io whenever the set changes so the renderer can re-fetch.

4. **`feat(web): add series detail route and panel`**
   - Route: `/series/:mangadexId` (source always MangaDex in v0.1).
   - `CoverCard` ŌåÆ links to `/series/:id`.
   - Layout: banner row with large cover (via `kirei-cover://mangadex/.../original`), title in Fraunces, author/artist/year/status in small caps meta row, follow button (bengara accent when followed), tags as hairline chips, synopsis in serif body clamp with "read more".
   - Hook: `useMangaDexSeries(mangadexId)` + `useMangaDexChapters(mangadexId, lang)`.
   - Chapter list: language filter dropdown (default EN), compact list with hairline dividers ŌĆö columns: `Vol`, `Ch`, title, group, relative date. Read-state badge placeholder (wired up in Slice E).

5. **`feat(web): wire follow toggle to library gateway`**
   - Hook `useFollow(mangadexId)` calling `library:follow` / `library:unfollow` over socket.
   - Renderer Zustand `libraryStore` caches followed set; re-fetches on `library:changed`.
   - Optimistic toggle with rollback on error.

6. **`test(desktop): integration test for follow/unfollow round-trip`**
   - In-memory SQLite. Mock MangaDex client. Follow ŌåÆ get ŌåÆ unfollow ŌåÆ get, assert state transitions.

## Slice D ŌĆö Reader

**Goal:** full-screen reader with page streaming from MangaDex @Home, keyboard nav, reading modes.

1. `feat(shared): add reader types` ŌĆö `ReaderMode = 'single'|'double'|'webtoon'`, `ReaderDirection = 'rtl'|'ltr'`, `FitMode = 'width'|'height'|'original'`, `ReaderSettings`.
2. `feat(desktop): implement mangadex.get-pages` ŌĆö wraps `MangaDexClient.getChapterPages(chapterId)`. Returns an array of `kirei-page://mangadex/{chapterId}/{filename}` URLs. Prefer `data` over `dataSaver` by default.
3. `feat(desktop): implement kirei-page protocol proxy (mangadex branch)` ŌĆö analogous to `kirei-cover`, but uses the rotating `baseUrl` from at-home response. Short TTL disk cache under `userData/pages/mangadex/{chapterId}/`. Atomic writes. Path validation. Note: at-home URLs expire ŌĆö refetch if a 404/403 comes back from cache read.
4. `feat(web): add reader route and page loader` ŌĆö route `/reader/:chapterId`. `useChapterPages(chapterId)` hook fetches page list, preloads next 3 pages with `<img>` prefetch.
5. `feat(web): implement single-page reader mode` ŌĆö fills viewport with letterboxing, fit-width/height toggle, zoom via wheel + pinch.
6. `feat(web): add keyboard navigation` ŌĆö ŌåÉ / ŌåÆ or A / D to move pages (respects RTL), Space/Enter for next, Home/End for first/last, F fullscreen, number keys for fit modes.
7. `feat(web): add double-page and webtoon modes` ŌĆö double auto-detects cover (single) then pairs from page 2. Webtoon = vertical scroll, no pagination, pages stacked with no gap.
8. `feat(web): add reader chrome` ŌĆö minimal top bar (back, chapter number, page indicator), auto-hides after 2s of inactivity. Preserves frameless Electron drag region.
9. `feat(web): add per-series reader preferences` ŌĆö reader mode, direction, fit mode stored per series in SQLite. Gateway channel `reader:set-prefs`.

## Slice F ŌĆö Offline chapter cache + update checks

1. `feat(desktop): download chapter command` ŌĆö `mangadex:download-chapter(chapterId)` fetches all pages via at-home and writes them into `userData/chapters/mangadex/{chapterId}/`. Streams progress events `download:progress`.
2. `feat(desktop): serve cached chapters when offline` ŌĆö `kirei-page://` protocol checks cache before refetching from at-home.
3. `feat(web): add download button to chapter list` ŌĆö per-chapter download state (not-downloaded / downloading / downloaded). Offline indicator in chapter row.
4. `feat(desktop): update checks for followed series` ŌĆö `mangadex:check-updates` polls `/manga/{id}/feed` for followed series, finds chapters newer than `lastCheckedAt`. Queue-aware (one request per second, batched).
5. `feat(web): new-chapter badges in library` ŌĆö number badge on library tiles when there are unread new chapters.
6. `feat(desktop): optional background update poller` ŌĆö on app start (and every 6h?) run `check-updates`, persist results, emit `library:updates-available`.

## Open questions to decide before shipping v0.1

- Do we ship bundled Fraunces/Geist/Shippori Mincho fonts (bigger installer, works offline) or keep them loading from Google Fonts (needs online on first launch)? Leaning bundle ŌĆö it's <1MB variable fonts and makes the app feel native.
- Cover cache size: cap at e.g. 500MB LRU, or unbounded until user clears? Default cap recommended.
- At-home URL expiry: how often to refetch? Start with "on 403/404 from cached fetch, refetch once."
- Should the app have any telemetry (even anonymous crash counts)? Default: none until explicitly added.
